# 社区刷题网站 —— 部署指南

> 技术栈:Next.js 14 + Supabase(Postgres + Auth）+ Vercel
> 域名:`sqtk.site`（**无需备案**,因为前后端都托管在海外平台)

---

## 〇、为什么这样部署?(关于备案的结论)

你的核心疑问是「域名没备案能不能用」。结论:

| 方案 | 能否用 sqtk.site | 说明 |
|------|------------------|------|
| 域名 + 腾讯云**国内**服务器 | ❌ 不能 | 国内服务器 80/443 端口需域名备案才能开放,这是工信部强制要求 |
| 域名 + **Vercel**(本方案) | ✅ 能 | Vercel 服务器在海外,**不受国内备案约束**,域名直接解析即可 |
| 域名 + 腾讯云**香港**服务器 | ✅ 能 | 海外节点同样无需备案 |

**本方案选择 Vercel + Supabase,完全不依赖你的腾讯云服务器,也不需要备案,域名可以立刻用起来。**
你的国内腾讯云服务器可以先闲置;等以后备案下来了,想迁回自建服务器也很容易(代码是标准 Next.js)。

## 二、第一步:创建 Supabase 项目(数据库 + 用户系统)

1. 登录 Supabase → 点击 **New Project**
2. 填写:
   - Name:`sqtk`(随意)
   - Database Password:**设一个强密码并记下来**(导入数据时不会用到,但要保管好)
   - Region:选 **Southeast Asia (Singapore)** 或 **Northeast Asia (Tokyo)**(离国内近)
3. 等 1-2 分钟项目创建完成。

### 2.1 建表(执行 SQL)

1. 左侧菜单进入 **SQL Editor** → **New query**
2. 打开项目里的 `supabase/schema.sql`,**全部复制**粘贴进去
3. 点 **Run** 运行。看到 Success 即建表完成(questions / profiles / attempts / wrong_questions / favorites 等表)。

### 2.2 拿到三个密钥

进入 **Project Settings(齿轮) → API**,记下:

| 名称 | 用途 | 对应环境变量 |
|------|------|--------------|
| Project URL | 项目地址 | `NEXT_PUBLIC_SUPABASE_URL` |
| `anon` `public` key | 前端用的公开密钥 | `NEXT_PUBLIC_SUPABASE_ANON_KEY` |
| `service_role` `secret` key | **仅导入题库用**,切勿泄露 | `SUPABASE_SERVICE_ROLE_KEY` |

### 2.3 关于邮箱验证(重要)

默认 Supabase 注册需要邮箱验证。备考网站想让用户「注册即用」,可以关掉:

- **Authentication → Providers → Email** → 关闭 **Confirm email**(Enable email confirmations 取消勾选)→ Save

这样用户填邮箱+密码即可直接登录,无需收验证邮件。
(若保留验证,代码也已兼容,会提示用户去邮箱确认。)

## 三、第二步:导入题库(3097 道题)

在**你自己的电脑**上操作一次即可(以后加题再跑一次):

```bash
# 1. 进入项目目录
cd 项目目录

# 2. 安装依赖(若还没装)
npm install

# 3. 创建 .env 文件(复制 .env.example 改名),填入:
#    NEXT_PUBLIC_SUPABASE_URL=你的 Project URL
#    SUPABASE_SERVICE_ROLE_KEY=你的 service_role key

# 4. 确认题库文件在 scripts/questions.json(已自动放好)
#    以后题库更新,替换这个文件即可

# 5. 运行导入
npm run import:questions
```

看到 `✅ 题库导入完成!` 即成功。脚本会自动:
- 去重、跳过空题干
- 给每题打 `quality` 标记:
  - **answerable(可判分)**:有答案+有选项的题,进入刷题判分
  - **view_only(仅浏览)**:PDF 解析残缺、缺答案的题,只在题库浏览里展示
- 当前数据约 **1267 题可判分**,其余作为浏览资料保留

> 💡 想提升可判分题量,只需把 `scripts/questions.json` 里残缺题目的 `answer`/`options` 补全,重新运行导入即可(脚本是 upsert,不会重复)。

## 五、第四步:Vercel 部署

1. 登录 Vercel → **Add New → Project** → 选中刚推送的 GitHub 仓库 → **Import**
2. Framework 会自动识别为 **Next.js**,无需改动
3. 展开 **Environment Variables**,添加两个(导入用的 service_role **不要**加到 Vercel):

   | Name | Value |
   |------|-------|
   | `NEXT_PUBLIC_SUPABASE_URL` | 你的 Project URL |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 你的 anon public key |

4. 点 **Deploy**,等 1-2 分钟。完成后会给一个 `xxx.vercel.app` 的临时网址,点开就能用了。

---

## 六、第五步:绑定域名 sqtk.site(无需备案)

### 6.1 在 Vercel 添加域名

1. 进入项目 → **Settings → Domains**
2. 输入 `sqtk.site` → **Add**(同时建议把 `www.sqtk.site` 也加上)
3. Vercel 会显示需要配置的 DNS 记录,通常是:
   - 根域名 `sqtk.site` → 一条 **A 记录**指向 `76.76.21.21`
   - 或 **CNAME** 指向 `cname.vercel-dns.com`(www 用这个)

### 6.2 在腾讯云改 DNS 解析

> 你的域名在腾讯云,所以去腾讯云改解析。**注意:这一步是改 DNS 解析,不是备案,不需要备案!**

1. 登录腾讯云 → **DNSPod / 域名解析** → 找到 `sqtk.site`
2. 按 Vercel 给的记录添加:

   | 主机记录 | 记录类型 | 记录值 |
   |---------|---------|--------|
   | `@` | A | `76.76.21.21`(以 Vercel 实际显示为准) |
   | `www` | CNAME | `cname.vercel-dns.com` |

3. 保存。等 DNS 生效(几分钟到几十分钟)。
4. 回到 Vercel Domains 页,状态变绿(Valid Configuration)即成功。Vercel 会**自动签发免费 HTTPS 证书**。

完成后访问 `https://sqtk.site` 就是你的刷题网站了 ✅

## 八、常见问题

**Q:必须用 Vercel 吗?能不能以后搬到我的腾讯云服务器?**
A:可以。等域名备案后,在腾讯云服务器上 `npm run build && npm start`,用 Nginx 反代到 3000 端口即可。数据库(Supabase)无需动。

**Q:Supabase / Vercel 免费额度够吗?**
A:对一个社区考试刷题站完全够用。Supabase 免费 500MB 数据库 + 5万月活认证;Vercel 免费 100GB 流量/月。

**Q:为什么有些题不能判分只能看?**
A:原始题库由 PDF 解析而来,部分题目答案/选项缺失。这些被标为「仅供浏览」,补全 JSON 后重新导入即可转为可判分。
